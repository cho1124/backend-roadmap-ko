## 네트워크 · HTTP

| 용어 | 뜻 |
|---|---|
| **HTTP** | 요청/응답 규약. 텍스트 기반 |
| **TCP** | 순서 보장·유실 재전송이 되는 연결 |
| **UDP** | 빠르지만 보장 없음 |
| **TLS/SSL** | 암호화 계층. HTTPS의 S |
| **DNS** | 이름 → IP 변환 |
| **포트** | 한 기기 안에서 프로그램을 구분하는 번호 |
| **엔드포인트** | 요청을 받는 하나의 주소 |
| **페이로드** | 요청/응답의 본문 |
| **헤더** | 본문에 딸린 메타데이터 |
| **안전(safe)** | 서버 상태를 바꾸지 않는 메서드 (GET) |

| **CORS** | 브라우저가 스스로에게 거는 출처 제약 |
| **Keep-Alive** | TCP 연결 재사용 |
| **HOL 블로킹** | 앞선 요청이 뒤를 막는 현상 |
| **폴링 / 롱폴링 / WebSocket / SSE** | 서버 변화를 받아오는 방식들 |

## 데이터베이스

| 용어 | 뜻 |
|---|---|
| **스키마** | 테이블 구조 정의 |
| **행 / 레코드** | 데이터 한 줄 |
| **기본키(PK)** | 행을 유일하게 식별하는 컬럼 |
| **외래키(FK)** | 다른 테이블의 PK를 가리킴. 관계의 실체 |
| **정규화** | 중복 제거, 하나의 사실은 한 곳에만 |
| **비정규화** | 성능을 위해 의도적으로 중복 허용 |
| **인덱스** | 빠른 검색용 보조 구조 (B-Tree) |
| **Full Scan** | 전체를 처음부터 훑기 (느림) |
| **실행 계획** | DB가 쿼리를 어떻게 처리할지 보여주는 것 |
| **JOIN** | 여러 테이블을 엮어 조회 |
| **트랜잭션** | 여러 작업을 전부-또는-전무로 묶음 |
| **ACID** | 원자성·일관성·격리성·지속성 |
| **격리 수준** | 동시 트랜잭션이 서로를 얼마나 보는가 |
| **갱신 유실** | 동시 수정으로 한쪽 변경이 사라지는 현상 |
| **더티 리드** | 커밋 안 된 데이터를 읽음 |
| **팬텀 리드** | 같은 조건 조회인데 행 개수가 달라짐 |
| **낙관적 락 / 비관적 락** | 충돌을 나중에 감지 / 미리 잠금 |
| **데드락** | 서로를 기다려 멈춤 |
| **N+1 문제** | 목록 1번 + 항목마다 1번 = N+1 쿼리 |
| **커넥션 풀** | DB 연결을 재사용하는 저장소 |
| **마이그레이션** | 스키마 변경 이력을 파일로 관리 |
| **ORM** | 객체 ↔ 테이블 자동 매핑 |
| **쿼리 빌더** | 타입 안전하게 SQL을 조립하는 도구 |

## API 설계

| 용어 | 뜻 |
|---|---|
| **REST** | 리소스 중심의 HTTP 설계 규약 |
| **리소스** | API가 다루는 대상(명사) |
| **CRUD** | 생성·조회·수정·삭제 |
| **미들웨어** | 요청 처리 전후에 끼어드는 단계 |
| **핸들러 / 컨트롤러** | 요청을 실제로 처리하는 함수 |
| **DTO** | 전송용으로 정리한 데이터 형태 |
| **직렬화** | 객체 → 전송 형식(JSON) |
| **페이지네이션** | 목록을 나눠서 주기 |
| **오프셋 / 커서** | 몇 번째부터 / "여기 다음부터" 방식 |
| **과소 응답(under-fetching)** | 필요한 걸 얻으려면 여러 번 호출해야 함 |
| **과다 응답(over-fetching)** | 안 쓰는 필드까지 다 받음 |
| **버저닝** | 호환 깨지는 변경 시 버전을 분리 |
| **레이트 리밋** | 요청 횟수 제한 |
| **OpenAPI / Swagger** | API 명세 표준 + 문서 도구 |
| **GraphQL** | 클라이언트가 필요한 필드를 지정하는 쿼리 API |
| **gRPC** | 바이너리 RPC. 주로 서버 간 통신 |
| **웹훅** | 서버가 먼저 상대를 호출하는 방식 |

## 인증 · 보안

| 용어 | 뜻 |
|---|---|
| **인증(AuthN)** | 너 누구야 |
| **인가(AuthZ)** | 너 이거 해도 돼 |
| **세션** | 서버가 신원을 기억하는 방식 |
| **세션 ID** | 그 기억을 가리키는 무의미한 랜덤값 |
| **토큰** | 신원 정보를 담은 문자열 |
| **JWT** | 서명된 토큰 형식. **암호화가 아니다** |
| **쿠키** | 브라우저가 자동으로 첨부하는 저장소 (형식이 아니라 **운반 수단**) |
| **액세스 토큰 / 리프레시 토큰** | 짧은 수명 / 갱신용 긴 수명 |
| **해싱** | 되돌릴 수 없는 단방향 변환 |
| **솔트** | 해싱에 섞는 사용자별 랜덤값 |
| **bcrypt / scrypt / argon2** | 비밀번호 전용 느린 해시 |
| **OAuth2** | 위임 인가 프로토콜 |
| **OIDC** | OAuth2 위에 인증을 얹은 것 (소셜 로그인) |
| **XSS** | 내 사이트에서 공격자 스크립트가 실행됨 |
| **CSRF** | 남의 사이트가 내 권한으로 요청을 위조 |
| **SQL 인젝션** | 입력이 SQL 코드로 해석됨 |
| **IDOR** | ID만 바꿔 남의 데이터에 접근 |
| **계정 열거** | 에러 메시지 차이로 계정 존재 여부가 새는 것 |
| **CSP** | 실행 허용 스크립트 출처를 제한하는 헤더 |
| **신뢰 경계** | 여기부터는 못 믿는다는 선 |

| 용어 | 뜻 |
|---|---|
| **환경변수** | 외부에서 주입하는 설정 |

| **컨테이너 / 이미지** | 실행 환경째로 고정한 패키지 |
| **멀티스테이지 빌드** | 빌드용/실행용 단계를 나눠 이미지를 줄이는 기법 |
| **오케스트레이션** | 컨테이너 여러 개를 자동 관리 (k8s 등) |
| **리버스 프록시** | 앞단에서 요청을 받아 뒤로 넘기는 서버 |
| **로드밸런서** | 요청을 여러 인스턴스로 분산 |
| **수평 확장 / 수직 확장** | 대수를 늘림 / 사양을 올림 |

| **liveness / readiness** | 살아있나(재시작 판단) / 받을 준비 됐나(트래픽 판단) |
| **우아한 종료** | 처리 중 요청을 마치고 종료 |
| **무중단 배포** | 서비스 끊김 없이 버전 교체 |
| **롤링 / 블루그린 / 카나리** | 배포 전략들 |
| **CI / CD** | 지속적 통합 / 지속적 배포 |
| **관측 가능성** | 내부 상태를 밖에서 알 수 있는 성질 |
| **구조화 로깅** | JSON 등 기계가 읽을 수 있는 로그 |
| **메트릭** | 숫자 시계열 지표 |
| **트레이스 / traceId** | 한 요청의 전체 경로 / 그 식별자 |
| **p95 / p99** | 상위 5%·1%에 해당하는 느린 응답 시간 |
| **골든 시그널** | 지연·트래픽·에러·포화도 |
| **SLO / SLA** | 목표 수준 / 계약된 수준 |
| **캐시 무효화** | 낡은 캐시를 언제 버릴지 결정하는 문제 |
| **CDN** | 정적 파일을 사용자 가까운 곳에 배포 |

## 이 로드맵에서 반복되는 핵심 원리 5개

용어가 아니라 **원리**다. 이게 남으면 나머지는 찾아 쓰면 된다.

```
① stateless           상태를 프로세스 밖에 둔다 → 확장·재시작이 가능해진다
② 신뢰 경계           밖에서 온 것은 전부 검증한다 → 코드와 데이터를 섞지 않는다
③ 배치화             비용을 항목마다 치르지 않는다 → 루프 안의 await, N+1 쿼리
④ 실패는 정상 경로     타임아웃·재시도·멱등성을 처음부터 설계한다
⑤ 관측 없으면 없는 것   로그·메트릭·트레이스가 서버의 감각기관이다
```
