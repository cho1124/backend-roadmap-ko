# 백엔드 학습 로드맵 (한국어)

> 프로그래밍 경험은 있지만 **서버 개발은 처음인 사람**을 위한 6개월 백엔드 학습 자료
> 스택: **Node.js + TypeScript** · 분량: 주 5~7시간 기준 24주 (약 164시간)

---

## 이 자료가 다른 로드맵과 다른 점

백엔드는 "배울 게 많은" 분야가 아니라 **"배울 게 흩어져 있는"** 분야다.
HTTP·DB·인증·배포가 서로 무관한 파편처럼 느껴지는 게 진짜 어려움이다.

이 자료는 그 파편들을 **하나의 인과 사슬**로 엮는다.

```
요청이 들어온다 (HTTP)
  → 서버가 그걸 처리한다 (런타임/동시성)
    → 상태는 밖에 저장한다 (DB)
      → 그 처리를 계약으로 노출한다 (API 설계)
        → 아무나 못 하게 막는다 (인증/보안)
          → 계속 살아있게 만든다 (배포/운영)
```

각 단계는 **앞 단계에서 생긴 문제를 해결하려고 태어난 기술**이다.
그래서 이 자료는 기술 목록이 아니라 **"왜 이렇게 됐는가"**를 축으로 쓰였다.

### 세 가지 원칙

| 원칙 | 내용 |
|---|---|
| **프레임워크를 먼저 배우지 않는다** | Stage 4까지 `node:http`만 쓴다. 불편을 겪은 뒤에야 프레임워크가 의미를 갖는다 |
| **완료 기준은 "말로 설명할 수 있는가"** | "돌아간다"는 기준이 아니다. 복붙해도 돌아간다. 각 문서 끝에 자가 점검 질문이 있다 |
| **오래 남을 것부터 배운다** | HTTP·SQL·동시성(20~40년) → 프레임워크·ORM(2~5년) 순서 |

## 신선도에 대하여

**11번 문서에만 유효기간이 있다.**

버전과 도구 선택은 빠르게 낡는다. 그래서 그런 정보를 나머지 문서에 섞지 않고
**11번 한 곳에만 격리**했다. 만료되면 11번만 다시 조사해서 갱신하면 되고,
나머지 11개(HTTP·이벤트 루프·트랜잭션·API 설계·인증 원리·관측 가능성)는 그대로 유효하다.

| | |
|---|---|
| 11번 조사일 | **2026-07-23** |
| 재검증 권장 | **2027-01** |
| 나머지 문서 | 감가상각 거의 없음 |

11번 문서의 모든 주장에는 출처 링크가 달려 있다.

## 이 로드맵이 도달하는 곳

> 6개월 164시간으로 도달하는 곳은 "백엔드 개발자"가 아니라
> **"백엔드 이야기가 통하는 사람"**이다.
>
> 서버 팀의 설계 회의에서 무슨 말인지 알아듣고, 왜 그렇게 하는지 질문할 수 있고,
> 필요하면 작은 API 하나는 스스로 만들어 올릴 수 있는 상태.

쿠버네티스와 카프카는 이 로드맵의 목적지가 아니다. 그래도 된다.
애플리케이션 백엔드의 기반을 모른 채 그 위를 배우면 아무것도 남지 않는다.

## 사용 방법

1. [01번 문서](01-백엔드의-5가지-근본-전제.md)를 먼저 읽는다 (기술이 아니라 사고방식에 관한 문서)
2. [02번 로드맵](02-6개월-로드맵.md)에서 자기 페이스에 맞게 기간을 조정한다
   - 주 5~7시간 → 6개월 (기본)
   - 주 10~14시간 → 3~4개월
   - 주 3시간 이하 → 1년
   - **순서는 바꾸지 않는다**
3. 각 스테이지를 순서대로, 마이크로 실습을 손으로 짜면서 진행한다
4. 스테이지 끝의 자가 점검 질문에 답할 수 없으면 그 스테이지는 안 끝난 것이다

## 유지보수 (작성자용)

문서는 Obsidian 볼트에서 쓰고 이 저장소로 밀어넣는다.
양쪽 파일은 **같은 포맷·같은 파일명**이라 보통은 단순 복사지만,
Obsidian이 링크를 `[[위키링크]]`로, 태그를 `#태그` 줄로 되돌려놓는 일이 있어
동기화 스크립트가 그 두 가지를 GitHub 포맷으로 정규화한다.

```bash
node sync.mjs --check   # 반영하지 않고 차이만 확인
node sync.mjs           # 볼트 → 저장소 반영
```

볼트 경로는 `BACKEND_VAULT` 환경변수로 덮어쓸 수 있다.
스크립트는 반영 후 **모든 문서 간 링크가 실제로 존재하는지 검증**하고,
깨진 링크나 알 수 없는 위키링크가 있으면 0이 아닌 코드로 종료한다.

> `#태그` 줄을 그대로 두면 GitHub이 **H1 제목으로 렌더링**한다.
> 문서마다 맨 아래에 거대한 제목이 하나씩 붙으므로 정규화가 필요하다.
